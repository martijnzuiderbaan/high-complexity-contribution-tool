# V1 Package Index

This index explains the current v1 Markdown package for the **High-Complexity
Contribution Tool** / **Instrument Complexiteit naar Bijdrage**.

The package is workbook-first. It contains practical Markdown artifacts for
reflection, manager preparation, facilitated conversation, interpretation, and a
bounded 30-day role-design experiment.

It is not a software application, diagnostic assessment, scoring model, hiring
tool, promotion tool, performance-review instrument, clinical tool,
occupational-health instrument, disability determination, or employee-ranking
system.

## Recommended Order of Use

## For an Adult Working Alone

1. Read the relevant boundary sections in `README.md` or
   `docs/USER_GUIDE.md`.
2. Complete the private parts of `docs/WORKBOOK_v1.md`.
3. Use the conversation preparation section to draft a minimum useful request.
4. Optionally use `docs/INTERPRETATION_GUIDE_v1.md` to check for
   over-interpretation.
5. Decide what to keep private before sharing anything.

## For an Adult Preparing a Manager Conversation

1. Read `docs/USER_GUIDE.md`.
2. Complete the current work situation, contribution pattern, and selected fit
   sections in `docs/WORKBOOK_v1.md`.
3. Prepare the conversation summary and disclosure boundaries.
4. Share only the relevant summary or examples, not the whole workbook unless
   that is safe and wanted.
5. Use `docs/MANAGER_CONVERSATION_GUIDE_v1.md` for the actual conversation.

## For a Manager or Employer

1. Read the boundaries in `README.md` and `docs/USER_GUIDE.md`.
2. Review `docs/MANAGER_CONVERSATION_GUIDE_v1.md`.
3. Prepare manager notes using concrete role/context examples.
4. Use `docs/WORKBOOK_v1.md` only for work-context fields and shared sections.
5. Agree on one 30-day experiment and review it after 30 days.

## For a Coach, HR Partner, or Facilitator

1. Read `docs/USER_GUIDE.md`.
2. Use `docs/WORKBOOK_v1.md` for adult preparation and joint work.
3. Use `docs/INTERPRETATION_GUIDE_v1.md` to keep interpretation bounded.
4. Use `docs/MANAGER_CONVERSATION_GUIDE_v1.md` to run the session.
5. Document only what is necessary for the agreed role-design experiment.
6. Pause or refer elsewhere if the issue is outside the tool's scope.

## For Product Development

1. Read `docs/v1_audit.md`.
2. Read `docs/PRD_v1.md`.
3. Compare the v1 tool pack against the three v0.1 legacy/reference artifacts
   only when product-development comparison is needed.
4. Use this index and `docs/PRIVATE_BETA_BLOCKER_FIXES.md` to decide what still
   needs to be created before larger beta waves or launch.

## What Is Still Missing for Private Beta

The Step 15 private-beta blocker pass is recorded in
`docs/PRIVATE_BETA_BLOCKER_FIXES.md`. Remaining private-beta work should stay
limited to invite-only operations unless a future step explicitly opens public
intake.

Before larger or more facilitated beta waves, the package still needs:

- Facilitator guide with session scripts, timing, and intervention points if
  the manager guide is not enough.
- 30-day experiment tracker as a separate artifact or printable handout.
- Content QA checklist for non-diagnostic boundaries, privacy, terminology,
  links, and frontmatter.
- Decision on language strategy: English-first, Dutch-first, or fully bilingual
  `nl-en`.
- Export review if PDF, DOCX, or printable editions are used.

Private beta should not collect completed personal scans. Invite-only contact,
consent, feedback, deletion, correction, and accidental-disclosure handling
should follow `docs/PRIVATE_BETA_RECORDS_SOP.md`.

## What Is Still Missing for Commercial Launch

Before commercial launch, the package still needs:

- Finalized public/commercial workbook package and public/commercial release
  notes.
- Product terms of use and license language.
- Privacy notice for website, downloads, purchases, feedback, and any services.
- Employer responsible-use terms that explicitly prohibit hiring, selection,
  promotion, performance review, ranking, diagnosis, and support-entitlement
  use.
- Clear guidance for completed workbook storage, retention, access, and
  deletion.
- Website or landing page copy.
- Free starter artifact, such as a mini-canvas or conversation checklist.
- Sample pages or previews.
- FAQ covering giftedness labels, privacy, HR misuse, burnout, accommodation,
  and when not to use the tool.
- Payment or download flow that does not collect completed personal responses.
- Support route and misuse reporting route.
- Legal and privacy review before collecting leads, purchases, feedback, or
  service inquiries.
- Evidence from private beta or expert review before making stronger claims.

Commercial launch should avoid claims that the tool is validated, diagnostic,
predictive, normed, clinically reviewed, legally reviewed, or scientifically
proven unless that evidence actually exists and has been reviewed.

# Open Items

## 1. Purpose

This file captures recommendations, unresolved items, and future steps that
should not interrupt the current planned step.

It is the persistent backlog for useful work that has been identified but not
yet scheduled. It helps keep each Codex step focused while preserving product,
privacy, commercial, and responsible-use follow-up work.

## 3. Maintenance rule

Future Codex steps should update this file when they identify useful work that
is not part of the current step.

explicitly asked. Add those items here instead, with a clear source step,
reason, timing, and status.

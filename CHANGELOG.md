# Changelog

## 1. Changelog Purpose

This changelog tracks meaningful product, documentation, release, safety, and
packaging changes for the **High-Complexity Contribution Tool** /
**Instrument Complexiteit naar Bijdrage**.

It is not a validation record, public launch announcement, commercial proof
record, legal review, privacy review, or production deployment log.

## 3. Private Beta v0.1.0 - 2026-06-22

Private Beta v0.1.0 is an invite-only Markdown/static-site release package for
trusted review and small manual beta use. It is not a public launch, paid
product launch, employer rollout, validated assessment, or production software
release.

### Added

- Added the initial product audit and v1 PRD to define the workbook-first
  scope, target users, responsible-use boundaries, launch risks, and
  non-diagnostic product direction.
- Added a README/product guide that explains the tool purpose, audience,
  workflow, package structure, commercial direction, status, and safety
  boundaries.
- Added the v1 core tool pack: workbook, interpretation guide, manager
  conversation guide, and package index.
- Added a dependency-free static landing page prototype in `site/` with local
  links to the Markdown package and no live intake.
- Added monetization, access/accounts, marketing, analytics, security, terms,
  privacy, responsible-use, trust/risk, beta, onboarding, demo, and example
  source documents.
- Added beta disclosure and consent materials for private, manual,
  invitation-based use.
- Added a private beta records SOP for manual contact records, consent logging,
  feedback notes, retention, correction, deletion, and accidental sensitive
  disclosure handling.
- Added synthetic examples for an adult workflow, manager conversation, and
  30-day role-design experiment.
- Added launch readiness review and private-beta blocker fix record.
- Added this private-beta release package: release notes, roadmap, launch
  checklist, documentation index, and changelog.

### Changed

- Made the v1 workbook, interpretation guide, manager guide, user guide,
  onboarding guide, package index, and beta operations documents the current
  private-beta entry path.
- Marked the root v0.1 adult scan, employer scan, and joint canvas as
  legacy/reference materials rather than the current private-beta route.
- Clarified that private beta access is invite-only and manual, with a direct
  private contact route agreed outside the static site.
- Reframed commercial, pricing, and launch wording as hypotheses to test
  during controlled beta, not as live offers or commercial validation.
- Kept analytics at optional local no-op/debug instrumentation only; no
  external analytics provider is connected.

### Fixed

- Fixed private-beta blocker `PVB-001` for invite-only use by defining manual
  contact routing through the invitation rather than the static site.
- Fixed private-beta blocker `PVB-002` by adding the private beta records SOP.
- Fixed private-beta blocker `PVB-003` by removing live-looking input fields
  from the static beta area and keeping no-submit/no-store language visible.
- Fixed private-beta blocker `PVB-004` by aligning current navigation toward
  the v1 package and labeling v0.1 files as legacy/reference.
- Fixed private-beta blocker `PVB-005` for private-beta sharing by replacing
  overclaiming launch wording with test, learn, bounded, exploratory, and
  commercial-learning language where needed.

### Documentation

- Added source documents for product direction, launch readiness, trust/risk,
  beta operations, manual records, access/accounts, monetization, analytics,
  marketing, privacy, security, terms, responsible use, onboarding, demo flow,
  and synthetic examples.
- Added package navigation through `README.md`, `docs/V1_PACKAGE_INDEX.md`,
  `docs/DOCUMENTATION_INDEX.md`, and `site/README.md`.
- Added release notes and checklist materials so a maintainer can send a
  coherent private-beta package to trusted reviewers without implying public
  launch readiness.

### Safety and Responsible Use

- Repeated the non-diagnostic, non-selection, non-performance-management, and
  non-ranking boundaries across user, manager, beta, privacy, security, terms,
  responsible-use, launch, and site materials.
- Preserved adult control over what private information is kept private or
  shared.
- Kept employer-side use focused on role/context conditions and one bounded
  30-day experiment.
- Prohibited collecting completed workbook responses, health details, HR case
  details, conflict narratives, third-party personal data, employee lists, or
  identifiable workplace stories as beta feedback.
- Kept the static site free of accounts, payments, backend APIs, databases,
  cookies, tracking scripts, third-party services, live form submission, and
  external analytics.

### Known Limitations

- The package is evidence-informed but not validated as a commercial
  assessment, clinical instrument, psychometric tool, or outcome predictor.
- The release is private beta only; it is not ready for public launch,
  self-serve sales, public data collection, paid checkout, broad employer
  rollout, or production deployment.
- No accounts, payments, backend, database, live form submission, external
  analytics provider, cookie banner, support portal, or download delivery flow
  exists.
- Privacy, security, and terms materials are placeholders for beta orientation
  and planning; they are not final legal, privacy, security, refund, license,
  or employer-pilot materials.
- Employer pilots remain gated by written scope, voluntary participation,
  adult control over sharing, no adverse consequences, data boundaries, and
  responsible-use controls.
- Facilitated beta still needs stronger scripts, a separate tracker/review
  handout, and facilitator materials before larger use.
